# Purpose

This document records known technical debt within Project Liminal.

Technical debt is any deliberate compromise, temporary solution, postponed improvement, or architectural limitation that should be revisited in the future.

Recording technical debt ensures these items are visible, prioritised, and resolved intentionally rather than forgotten.

---

# When to Add Technical Debt

Create a Technical Debt entry whenever:

- A temporary implementation is chosen over the ideal solution.
- A feature is deliberately postponed.
- Performance is knowingly sacrificed.
- Code duplication is accepted temporarily.
- Refactoring is intentionally deferred.
- A third-party limitation affects the project.

---

# Priority Levels

### Critical

Must be resolved before continuing development.

### High

Should be resolved within the next few sprints.

### Medium

Can remain temporarily but should be monitored.

### Low

Acceptable until a suitable opportunity for improvement.
