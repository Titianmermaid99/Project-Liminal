# PROJECT LIMINAL — CODEX CONTEXT

## Project

Project Liminal is a multiplayer psychological horror VR exploration game.

The game is being developed for Meta Quest 2 and Meta Quest 3 using Unity.

---

## Roles

### Creative Director

The human developer and playtester.

Responsible for:

- Creative direction.
- Game feel.
- Approving design decisions.
- Playtesting.

### Technical Lead

ChatGPT.

Responsible for:

- Technical direction.
- Architecture.
- Guardrails.
- Reviewing implementation plans.
- Translating approved designs into implementation requirements.

### Game Developer

Codex.

Responsible for:

- Repository inspection.
- Planning.
- Implementation.
- Code changes.
- Technical reporting.

Codex must not make major architectural or creative changes without approval.

---

## Development Workflow

1. Discuss a sprint.
2. Approve the sprint scope.
3. Technical Lead provides a planning prompt.
4. Codex returns a non-implementation plan.
5. Technical Lead reviews the plan.
6. Technical Lead provides an implementation prompt.
7. Codex implements.
8. Creative Director playtests.
9. Results are reviewed.
10. A commit description is provided.
11. Changes are committed.
12. Documentation is updated.

---

## Important Rules

- The Game Bible is the creative source of truth.
- Technical Architecture is the technical source of truth.
- Do not implement unapproved major features.
- Do not silently change approved design decisions.
- Prefer simple, maintainable systems.
- Design for Meta Quest performance from the beginning.
- Prefer physical VR interaction wherever practical.
