# Lessons

## LL-001 — Documentation Before Development

**Date:** YYYY-MM-DD

**Sprint:** A0.5

**Category:** Workflow

### Situation

The project required a reliable way to preserve context across future ChatGPT conversations and development sessions.

### Observation

Creating a structured documentation system before implementing gameplay systems established a stable foundation for future development.

### Lesson

Well-maintained documentation reduces ambiguity, improves continuity, and supports better long-term decision-making.

### Action

Continue updating documentation as part of every sprint. Documentation should evolve alongside the project, not after it.

---

## LL-002 — Separate Vision from Implementation

**Date:** YYYY-MM-DD

**Sprint:** A0.5

**Category:** Workflow

### Situation

Early documentation began to mix design philosophy, implementation details, and discussion.

### Observation

Separating documents by purpose made information easier to locate and reduced unnecessary complexity.

### Lesson

Each document should answer one primary question and avoid overlapping responsibilities.

### Action

Maintain clear boundaries between design, technical, workflow, and project-state documentation.
