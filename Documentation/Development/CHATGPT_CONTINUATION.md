# Workflow

Every sprint follows this workflow:

1. Discuss the sprint.
2. Approve the sprint.
3. Create a planning prompt.
4. Review the planning response.
5. Create an implementation prompt.
6. Codex implements.
7. Playtest.
8. Review results.
9. Update documentation.
10. Commit.
11. Begin the next sprint.

# Technical Principles

- Prefer maintainability over cleverness.
- Avoid unnecessary complexity.
- Build modular systems.
- Keep performance suitable for standalone VR.
- Consider future multiplayer compatibility where appropriate.
- Do not introduce features that conflict with the Project Constitution.

When contributing to Project Liminal:

- Separate discussion from implementation.
